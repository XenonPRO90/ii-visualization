# Курс «Claude Code» от aizdec.me — полный конспект

Спарсено 2026-04-22 с https://aizdec.me/courses/claude-code (все 21 урок + главная).
Сырые страницы (markdown) лежат в `docs/aizdec_raw/`.

## 1. Обзор курса

- **Название:** Claude Code (платформа ИИздец)
- **Автор/школа:** ИИздец (aizdec.me), автор — **Том**
- **ЦА:** Люди без опыта программирования, которые хотят создавать проекты (сайты, боты, приложения)
- **Формат:** Бесплатный курс на платформе, текст + примеры, 21 урок
- **Ключевые инструменты:** Claude Code, VS Code, GitHub + Git, AquaVoice (опц.), Node.js/npm (localhost)
- **Главная идея:** Claude Code — не советчик как ChatGPT, а исполнитель. Видит структуру проекта, работает параллельно, держит ~200k токенов контекста.

## 2. Все 21 урок (по порядку на сайте)

### Модуль: УСТАНОВКА
1. **what-is-claude-code** — Что такое Claude Code. Отличие от ChatGPT, что умеет, требования (Pro/Max подписка).
2. **install-vscode** — Установка VS Code на Mac/Win (из code.visualstudio.com, обязательно в PATH на Windows).
3. **install-claude-code** — Установка расширения Claude Code в VS Code. Авторизация Anthropic. Важно: VPN в РФ.
4. **first-project** — Создание папки `Документы/Projects/my-first-project` (без пробелов/кириллицы), Open Folder.
5. **localhost** — Локальный сервер через npm. Локальный адрес + сетевой, hot reload, Ctrl+C для остановки.

### Модуль: GitHub и основы
6. **github-setup** — Регистрация GitHub, Sign In в VS Code, создание репо (Public/Private), Publish to GitHub.
7. **interface** — Интерфейс Claude Code: типы ответов (текст / файл / diff), слэш-команды (`/clear`, `/compact`, `/attach`), история чатов (Resume conversation), индикатор контекста.
8. **how-to-talk** — Правила общения: конкретность, итерации, контекст, один чат. Чего избегать: слишком общие запросы, слишком много задач сразу.

### Модуль: ДОКУМЕНТАЦИЯ (критично по мнению автора)
9. **why-documentation** — Зачем план перед кодом. Два типа: проектная (всё) и техническая (фича). Структура: Название / Что это / Для кого / Функции / Страницы / Чего НЕ будет / Технические заметки.
10. **writing-docs** — Как писать вместе с Claude. Надиктовать идею (AquaVoice) → попросить структурировать → найти функции → отобрать нужное → сохранить в `docs/project.md` или `README.md`.
11. **improvement-cycle** — Цикл улучшения: Перечитать → «Ты уверен?» → Критический взгляд (как senior dev) → Пересмотреть. 2-3 прохода, потом стоп.

### Модуль: РАБОТА С ФАЙЛАМИ
12. **files** — Claude создаёт/читает/меняет/удаляет файлы. Чтение diff: красный минус = удаление, зелёный плюс = добавление. Всегда читать diff перед Accept.
13. **attach** — Скриншоты (Cmd+Shift+4 / Win+Shift+S) + `/attach` или вставка в чат. Скриншот + текст = лучший результат. Навигация по проекту через Claude.
14. **git-basics** — Commit = сохранился на компе, Push = в облако. Частые маленькие коммиты. Откат: «Покажи историю» → «Верни к коммиту». Типичные грабли: забыл commit / забыл push / закоммитил нерабочее.

### Модуль: ДЛЯ ПРОДВИНУТЫХ
15. **research** — Режимы исследования и критики. Роли («как senior dev», «как тестировщик», «как хакер»). Переспрашивание: «Ты уверен?», «Какие минусы?», «Что может пойти не так?». План 3-5 шагов перед действием.
16. **multiple-chats** — Метод «три вкладки»: Чат 1 общение, Чат 2 работа, Чат 3 проверка. Новый чат: Shift+Cmd+Esc. Когда `/clear`, когда новый чат, когда продолжить.
17. **debugging** — Отладка. Не паниковать. Скопировать текст ошибки → «Что это значит?». Если не помогает — скриншот + описание. Консоль браузера. Универсальное решение: Developer: Reload Window.
18. **claude-md** — Файл CLAUDE.md (правила для Claude, он читает при каждом запуске). `/init` — базовый. Локации: корень, подпапки, CLAUDE.local.md (не в git), ~/.claude/CLAUDE.md (глобально). Правила NEVER/ALWAYS/MUST. До 500 строк.
19. **work-modes** — Три режима редактирования: **Plan Mode** (только думает), **Ask Before Edits** (safe, подтверждать), **Edit Automatically** (быстро, но рискованно). Когда какой. **Extended thinking** (иконка внизу) — думает дольше, глубже.
20. **troubleshooting** — Известные баги VS Code + Claude Code: серый экран, история не грузится, бесконечная загрузка, высокое потребление памяти. Профилактика: частые коммиты (15-30 мин), не торопиться, обновлять, /compact.
21. **whats-next** — Резюме. Два пути развития: (a) публикация проекта (домен, хостинг, БД, auth, платежи, боты); (b) тюнинг Claude Code (Skills, MCP, Hooks, Worktrees, Agent Teams).

## 3. Общие принципы автора Тома

1. **80% думай, 20% кодь** — документация и планирование экономят переделки
2. **Документация = карта**, без неё Claude создаёт «что-то»
3. **Контекст — король**: показывай файлы, не полагайся на «сам поймёт»
4. **Итерации > совершенство**: три коротких уточнения быстрее одного «идеального» запроса
5. **Параллелизм — суперсила**: три чата одновременно (общение / работа / проверка)
6. **Роль меняет результат**: «как senior dev» vs «как хакер» — разные ответы
7. **Безопасность по умолчанию**: Private репо, CLAUDE.md с запретами, частые коммиты
8. **Ошибки — это информация**, не катастрофа
9. **Не надо быть программистом** — достаточно объяснять словами

## 5. Стиль автора — что стоит перенять

- Честность про свои грабли: «я потерял дни работы потому что забыл про коммиты»
- Практические советы только из реального опыта, без теории ради теории
- Мотивирующий тон: «это нормально», «не паникуй», «обновления выходят регулярно»
- Реальные команды, реальные кнопки, реальные скриншоты
