Иногда нужно показать Claude что-то конкретное: скриншот ошибки, дизайн-макет, документ. В этом уроке — как прикреплять файлы и изображения.

[ СКРИНШОТЫ ]

Самый частый случай — показать ошибку или результат.




Как прикрепить:

Сделай скриншот (Cmd+Shift+4 на Mac, Win+Shift+S на Windows)

Вставь прямо в поле ввода (Cmd/Ctrl + V) — или используй команду /attach

Напиши сообщение и отправь

Claude увидит изображение и поймёт контекст.

[ ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ ]

Ошибка на экране:
Вот что показывает браузер. Почему не работает?
[скриншот ошибки]

Дизайн-макет:
Вот макет из Figma. Сделай похожую страницу.
[скриншот дизайна]

Результат который не нравится:
Получилось вот так, но нужно по-другому. Кнопка должна быть справа.
[скриншот текущего результата]


Совет:
Скриншот + текстовое описание = лучший результат. Claude видит картинку, но текст помогает понять что именно не так.

[ НАВИГАЦИЯ ПО ПРОЕКТУ ]

Claude умеет искать в проекте. Используй это.

Структура проекта:
Покажи структуру проекта
Какие папки и файлы есть в корне?

Поиск файлов:
Найди файл где определена функция calculateTotal
В каком файле находится компонент Header?

Поиск по содержимому:
Найди где используется функция sendEmail
Покажи все места где вызывается API /users

[ ПОНИМАНИЕ КОДА ]

Нашёл файл, но не понимаешь что там:

– Объясни что делает этот файл
– Как работает функция processOrder?
– Зачем нужен этот useEffect?
– Claude объяснит простым языком. Можно уточнять:
– А почему здесь async/await?
– Что будет если убрать эту проверку?

[ СВЯЗИ МЕЖДУ ФАЙЛАМИ ]

Проекты — это связанная система:

– Какие файлы импортируют utils/helpers.js?
– Откуда вызывается функция authenticate?
– Покажи цепочку от нажатия кнопки до запроса к серверу
– Claude покажет как части проекта связаны. Бесценно когда нужно понять как что-то работает.




⚠️ Важно:
В больших проектах такие запросы могут занять время. Claude читает много файлов. Подожди — результат стоит того.



[ БЫСТРЫЕ ЗАПРОСЫ ]

Найти точку входа:
– Какой файл запускается первым?
Найти конфигурацию:
– Где настройки базы данных?
Найти тесты:
– Где находятся тесты?
Найти стили:
– Где определены глобальные стили?

[ ПРАКТИКА ]

Если у тебя есть проект:

Сделай скриншот любого файла и отправь Claude с вопросом "что это?"

Попроси показать структуру проекта

Найди какой-нибудь файл по функции которая в нём есть

Если проекта нет — создай тестовую папку с несколькими файлами.




💭 От Тома:
Скриншоты — это кайф. Раньше описывал ошибки словами, тратил время. Теперь — скриншот + "почему не работает?" и Claude сразу видит проблему. Особенно полезно когда ошибка в консоли браузера или терминале.
