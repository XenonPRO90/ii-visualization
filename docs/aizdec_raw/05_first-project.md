Прежде чем что-то делать с Claude — нужно место где это делать. Проект. Папка на компьютере, которую откроешь в VS Code.

[ ГДЕ ХРАНИТЬ ]

Сначала создай место для будущих проектов:

Рекомендую:

Документы/Projects/ — папка для всех проектов

Внутри — отдельная папка для каждого проекта




Пример структуры:

Документы/
  Projects/
    my-first-project/    ← здесь будет твой первый проект
    telegram-bot/
    landing-page/
Копировать

Так всё будет в одном месте и не потеряется.

💡 Совет:
Создай папку Projects прямо сейчас. Всё остальное будем делать внутри неё.

[ ЧТО ТАКОЕ ПРОЕКТ ]

Несложно догадаться, что это просто папка с файлами. Всё что Claude создаст: код, стили, документы — сохраняется туда.

Почему это важно:

Порядок — все файлы проекта в одном месте

Контекст — Claude видит все файлы в папке и понимает связи

История — можно вернуться к старым версиям (об этом позже)

Один проект = одна папка = одна идея.

[ СОЗДАНИЕ ПРОЕКТА ]
Если не знаешь как, то на Mac:

Открой Finder (синяя иконка со смайликом)

Перейди в Документы (слева)

Нажми Cmd + Shift + N (или правой кнопкой → "Новая папка")

Назови папку, например: my-first-project

На Windows:

Открой Проводник (иконка папки)

Перейди в Документы

Нажми Ctrl + Shift + N (или правой кнопкой → "Создать папку")

Назови папку, например: my-first-project

💡 Совет:
Название без пробелов и русских букв. Используй дефис или подчёркивание:
✅ my-project
✅ task_manager
❌ мой проект
❌ My Project!!!

[ ОТКРЫТИЕ В VS CODE ]

Теперь открываем папку в VS Code.

Способ 1 — через VS Code:

Открой VS Code

File → Open Folder (или Cmd/Ctrl + O)

Найди свою папку и выбери её

Нажми "Open"

Способ 2 — перетаскиванием:

Открой VS Code

Перетащи папку прямо на окно VS Code

Способ 3 — из Finder/Проводника:

Правой кнопкой на папке

"Открыть с помощью" → VS Code

[ ЧТО ТЫ ВИДИШЬ ]

После открытия папки:

Слева — панель файлов:

Название твоей папки сверху

Пока там пусто — это нормально

Здесь будут появляться файлы которые создаст Claude

Справа — пустая область:

Это место для открытых файлов

Сейчас там ничего — файлов ведь нет

Внизу — строка состояния:

Показывает что VS Code готов к работе

⚠️ Важно:
Убедись что папка открыта как проект. Сверху в заголовке окна должно быть имя твоей папки. Если там написано "Welcome" или пустое — повтори шаги открытия.

[ ПРОВЕРКА CLAUDE CODE ]

Теперь проверим что Claude Code работает в этой папке.

Открой панель Claude Code:

Быстрый способ: Cmd/Ctrl + Shift + P → напиши "Claude Code: Open in sidebar" → Enter

Или: найди иконку Claude (оранжевый логотип) в боковой панели и кликни

Справа откроется панель чата. Если видишь поле для ввода сообщений — всё работает.

⚠️ Если панель не открывается:
1. Проверь что расширение установлено (урок 2)
2. Перезапусти VS Code полностью
3. Попробуй снова через Command Palette
Если всё равно не работает — вернись к разделу "Если что-то не работает" в уроке 2.

[ ПЕРВЫЙ ЗАПРОС ]

Давай попробуем. В поле ввода напиши:

Привет! Ты меня слышишь?

Нажми Enter.

Claude ответит. Это значит расширение работает и можно начинать.

💡 Важно:
Claude ответит даже если папка не открыта. Но для работы с файлами нужен проект. Проверь что слева в интерфейсе VS Code видно имя твоей папки — это значит проект открыт.

💭 От Тома:
Не парься пока что папка пустая. Это правильно. Claude будет создавать файлы по твоим запросам. Сначала думаем что делать, потом делаем. А чтобы начать "думать" файлы не нужны, нужен только чат.

[ ПРОВЕРКА ]

✅ Создал папку для проекта

✅ Открыл её в VS Code

✅ Вижу панель файлов слева (пока пустую)

✅ Claude отвечает на сообщения

Всё так? → Модуль 1 завершён! Базовая настройка готова.




💭 И ещё от Тома:
Работа начинается не сразу. Сначала мы с агентами часами обсуждаем идею, потом Claude создаёт структуру. И только потом, когда готова вся необходимая документация, я перехожу к делу. Вернее, они переходят, а я наблюдаю :)
