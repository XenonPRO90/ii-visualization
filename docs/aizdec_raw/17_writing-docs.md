Открой Claude Code и просто расскажи свою идею. Как другу. Без структуры.

Пример:

Хочу сделать телеграм-бота для своей кофейни. Чтобы люди могли смотреть меню и делать предзаказ. Ну типа выбираешь напиток, говоришь когда заберёшь, приходишь — уже готово. Ещё хочу чтобы была программа лояльности, каждый 5-й кофе бесплатно.




Попроси структурировать:

Помоги превратить это в документацию проекта. Сделай структуру: что это, для кого, основные функции.

Братишка создаст структурированный документ.

[ ШАГ 2: ПОПРОСИТЬ НАЙТИ ИНФО ]

Claude знает много. Используй это.

Какие функции обычно есть в ботах для кофеен? Что я мог упустить?

Какие есть варианты программ лояльности кроме "каждый N-й бесплатно"?

Как технически делают предзаказы? Что нужно знать заранее?

Claude найдёт информацию и предложит идеи. Ты выбираешь что подходит.

💡 Совет:
Не принимай всё что он говорит. Пока это только предложения. Ты можешь их отклонять, выбирать что нравится больше. И ты можешь заставлять его самого сомневаться в своих идеях, прежде чем переходить к их реализации.

Создай файл README.md с описанием проекта

[ ПРИМЕР ДИАЛОГА ]

Вот как может выглядеть весь процесс:

Ты: Хочу сделать сайт-портфолио для себя как фрилансера. Я делаю дизайн и иллюстрации. Нужно показать работы и чтобы люди могли связаться.

Claude: задаёт уточняющие вопросы или сразу предлагает структуру

Ты: Да, примерно так. Добавь раздел с ценами. И убери блог — не буду вести.

Claude: обновляет структуру

Ты: Какие разделы обычно есть на портфолио дизайнеров? Что я мог забыть?

Claude: предлагает идеи: отзывы, процесс работы, FAQ

Ты: Отзывы хорошая идея. Добавь. Остальное не надо.

Claude: добавляет

Ты: Создай файл docs/project.md с этой документацией

Claude: создаёт файл

Вот что обычно включает документация проекта:

## Для кого
Кто будет пользоваться? Целевая аудитория.

## Проблема
Какую проблему решает? Зачем это нужно?

## Основные функции

[ ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ ]

Когда начнёшь разрабатывать, перед каждой новой фичей полезно писать техническую документацию.

Это выглядит так:

Хочу добавить программу лояльности в бота. Помоги разобраться: как это обычно делают? Какие есть варианты? Как хранить баллы пользователей?

Claude найдёт информацию, расскажет про варианты, предложит решения.

Потом просишь оформить:

Создай файл docs/features/loyalty-program.md с описанием того что мы решили сделать

Теперь у тебя есть документ по конкретной фиче. Когда будешь делать — всё под рукой.

💡 Когда писать техническую документацию:
- Перед добавлением новой функции
- Когда задача сложная и непонятно как подступиться
- Когда нужно изучить как это делают другие

Когда НЕ нужно:
- Мелкие правки и баги
- Очевидные изменения
- Когда уже всё понятно

[ ПРАКТИКА ]

Время сделать документацию для своего проекта.

Открой Claude Code

Расскажи идею — просто своими словами

Попроси структурировать — пусть Claude оформит

Спроси что упустил — "какие функции ещё бывают?"

Выбери нужное — не всё, только что реально нужно

Сохрани в файл — docs/project.md или README.md

После этого у тебя будет документ. Основа будущей ракеты.

⚠️ Важно:
Не пытайся сделать идеально. Документация будет меняться по мере движения вперед, как и функциональность твоего проекта. Сейчас важно зафиксировать основу. И вбить себе новую привычку – сначала 80% времени (думать и) писать документацию, и только 20% оставшегося времени делать. Это оооочень сэкономит тебе нервы.
