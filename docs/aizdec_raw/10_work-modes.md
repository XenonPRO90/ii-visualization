Claude Code умеет работать по-разному: осторожно спрашивать каждый шаг или действовать автономно. Внизу панели есть переключатели режимов: разберёмся что они делают.

[ РЕЖИМЫ РЕДАКТИРОВАНИЯ ]

Внизу панели Claude Code есть кнопка режима. Там три варианта:

1. Plan Mode
В этом режиме, он ничё не делает, а только думает и планирует (ну и еще текстовые файлы планов может создавать и редактировать, но не код проекта).

2. Ask before edits

Claude показывает каждое изменение и ждёт твоего подтверждения. Самый безопасный режим — ты контролируешь всё.

3. Edit automatically

Claude сам применяет изменения без подтверждения. Быстрее, но рискованнее — он может сделать что-то не то, поэтому нужно быть осторожным.

💡 Какой выбрать:
- Начинающим — Ask Before Edit. Видишь всё что происходит.
- Когда доверяешь задаче — Auto Edit. Быстрее для рутины.

[ КОГДА ИСПОЛЬЗОВАТЬ АВТОМАТИЧЕСКИЙ РЕЖИМ ]

Edit automatically экономит время, но используй осторожно, особенно в первое вреся:

Хорошо для:

Простые задачи где сложно ошибиться

Массовое форматирование кода

Создание файлов по шаблону

Когда ты рядом и следишь

Плохо для:

Удаление файлов

Работа с конфигурациями

Первый раз делаешь что-то новое

Когда отошёл от компьютера

⚠️ Важно:
Даже в автоматическом режиме CLAUDE.md работает. Если там написано "всегда спрашивать перед удалением" — Claude спросит.

[ РЕЖИМ ДУМАНИЯ ]

Рядом с полем ввода есть иконка — это переключатель режима думания (extended thinking).

Обычный режим:

Claude отвечает быстро. Подходит для простых задач.

Расширенное думание (иконка активна):

Claude думает дольше, но глубже. Лучше для сложных задач: архитектура, отладка запутанных багов, планирование.

💡 Когда включать расширенное думание:
- Сложная задача с множеством файлов
- Нужно продумать архитектуру
- Баг который не можешь найти
- Хочешь более качественный ответ

Когда не нужно:
- Простые вопросы
- Создание одного файла
- Мелкие правки

[ КОМБИНИРОВАНИЕ РЕЖИМОВ ]

Режимы можно (нужно) комбинировать под задачу:

Рутинная работа:

Auto Edit + обычное думание

→ Быстро, не отвлекает

Сложная задача:

Ask Before Edit + расширенное думание

→ Качественно, под контролем

[ ВЫДЕЛЕНИЕ КОНТЕКСТА ]

Когда выделяешь код в редакторе — Claude видит что именно ты выделил. Внизу панели появляется "X lines selected".

Как работает:

Выдели код в редакторе (мышкой или Cmd/Ctrl + A для всего файла)

Этот код автоматически добавляется в контекст сообщения

Теперь можешь спросить про этот конкретный код

Зачем:

Показать конкретный кусок кода для исправления

Спросить про выделенный фрагмент

Дать контекст без копирования

💭 От Тома:
Выделение удобная штука. Вместо "найди функцию X и исправь", я иногда выделяю код и говорю "исправь это". Claude сразу видит что именно, о каких строках идет речь.




[ ПРАКТИКА ]

Найди переключатель режима внизу панели

Попробуй разные режимы на простой задаче

Включи расширенное думание и задай сложный вопрос

Выдели код в файле и спроси Claude про него

[ ПРОВЕРКА ]

✅ Знаю три режима редактирования

✅ Понимаю когда какой использовать

✅ Знаю про режим расширенного думания

✅ Умею выделять код для контекста




💭 От Тома:
Режим полного автопилота можно (и даже нужно) включать, но только если твоя система регулярно сохраняет промежуточный результат, и ты всегда можешь откатиться к недавней версии. 

Короче, давать агентам свободу – это ок. Давать им рисковать твоим проектом – это не ок.
