Проект создан. Но как увидеть что получается? Не в виде файлов, а как это выглядит на самом деле — в браузере, как настоящий сайт.

[ ЧТО ТАКОЕ LOCALHOST ]

Когда ты делаешь сайт, он должен где-то работать. В интернете сайты живут на серверах. Но пока ты разрабатываешь — сервер не нужен. Твой компьютер сам становится сервером.

localhost:8080 это адрес твоего локального сервера. Как обычный сайт, только доступен только тебе.

💡 Простыми словами:
Localhost это "мой компьютер". Число после двоеточия (8080) это порт, как номер комнаты. Разные проекты могут работать на разных портах. Если всё равно ничё не понял, спроси у Клода, он пояснит. 

[ КАК ЗАПУСТИТЬ ]

Claude сам запускает локальный сервер когда нужно. Просто попроси:

Запусти дев сервер / Запусти проект

Или:

npm run dev

Claude выполнит команду и покажет что-то типа:

Local:   http://localhost:8080/
Network: http://192.168.1.100:8080/
Копировать

Local — адрес для твоего компьютера

Network — адрес для других устройств в твоей сети

[ ОТКРЫВАЕМ В БРАУЗЕРЕ ]

Скопируй адрес http://localhost:8080 (или какой показал Claude)

Открой браузер (Chrome, Safari, Firefox — любой)

Вставь адрес в строку и нажми Enter

Готово. Ты видишь свой проект как настоящий сайт.

⚠️ Важно:
Пока сервер работает, сайт доступен. Закроешь терминал, остановишь сервер, выключишь комп – и сайт пропадёт.

[ СМОТРИМ С ТЕЛЕФОНА ]

Хочешь посмотреть проект на телефоне? Легко.

Условия:

Телефон и компьютер в одной Wi-Fi сети (!)

Сервер запущен

Как:

Найди адрес Network (например http://192.168.1.100:8080)

Открой этот адрес в браузере на телефон

Готово — видишь свой проект ✔️

💡 Совет:
Network-адрес можно отправить себе в телеграм или мессенджер — так быстрее открыть на телефоне чем набирать вручную. А, ну и спросить его тоже можно у агента. Просто скажи агенту, что хочешь посмотреть результат на мобиле.

[ ИЗМЕНЕНИЯ В РЕАЛЬНОМ ВРЕМЕНИ ]

Большинство современных проектов поддерживают hot reload:

Сервер запущен, сайт открыт в браузере

Claude меняет файл

Браузер автоматически обновляется

Не надо каждый раз нажимать F5. Изменил —> увидел. Это удобно для разработки.

[ ЕСЛИ ПОРТ ЗАНЯТ ]

Иногда видишь сообщение:

Port 8080 is in use, trying another one...
Local: http://localhost:8081/
Копировать

Это нормально. Значит на порту 8080 уже что-то работает (другой проект или программа). Сервер автоматически выбрал другой порт — 8081. Просто используй новый адрес. Так можно запускать одновременно несколько проектов.

[ ОСТАНОВКА СЕРВЕРА ]

Чтобы остановить сервер:

В терминале: нажми Ctrl + C

Или: закрой вкладку терминала

Или: попроси агента / перезагрузи комп.

После остановки localhost перестанет работать. Чтобы снова увидеть проект, запусти сервер заново.
